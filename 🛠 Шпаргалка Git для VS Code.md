### 1. Начало работы (Синхронизация)

- `git checkout main` — перейти на главную ветку (код).
    
- `git checkout docs` — перейти на ветку документации (Obsidian).
    
- **`git pull`** — **САМАЯ ВАЖНАЯ КОМАНДА.** Скачать все обновления от друга из облака. Делай это перед тем, как начнешь что-то писать.
    

### 2. Сохранение изменений

- `git add .` — подготовить все измененные файлы к сохранению.
    
- `git commit -m "описание"` — создать "точку сохранения" с комментарием.
    
- **`git push`** — отправить свои сохранения на GitHub, чтобы друг их увидел.
    

### 3. Работа с ветками (Фичи)

- `git checkout -b feat-name` — создать новую ветку для теста/фичи и сразу перейти в неё.
    
- `git branch` — посмотреть список всех веток (та, что со звездочкой — активна).
    
- `git merge feat-name` — (находясь в main) влить в себя изменения из тестовой ветки.
    

### 4. Если что-то пошло не так

- `git status` — подсказка от Гита: где ты, что изменил и что делать дальше.
    
- `git checkout .` — **ОПАСНО:** отменить все несохраненные изменения в файлах (вернуть как было до твоих правок).
