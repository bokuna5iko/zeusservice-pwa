1. **Pull (Стрелка вниз `⬇️` в кружке):**
    
    - **Обязательно нажимай её первой**, когда садишься за работу. Это скачает всё, что изменил твой друг.
        
2. **Stage (Плюс `+` рядом с файлом):**
    
    - Ты нажал на `+` у файла `Instruction`, и он переместился в **Staged Changes**. Это правильно — ты подготовил его к отправке.
        
    - _Совет:_ Файл `workspace.json` лучше вообще не пушить (это чисто твои настройки окна Obsidian), его можно оставить в `Changes`.
        
3. **Commit (Галочка `✔️` в кружке):**
    
    - В поле, где у тебя написано `vault backup: {{date}}`, лучше писать нормальное сообщение, например: `docs: обновил общие инструкции`.
        
    - Нажимай галочку. Теперь твои изменения «упакованы» локально.
        
4. **Push (Стрелка вверх `⬆️` в кружке):**
    
    - Это финальный аккорд. Нажимай её, и твои изменения улетят на GitHub.
        

---

### Что означают буквы рядом с файлами на твоем скрине:

- **M (Modified):** Файл изменен (как твой `Instruction`).
    
- **U (Untracked):** Новый файл, который Git еще не знает.
    
- **D (Deleted):** Файл был удален.
