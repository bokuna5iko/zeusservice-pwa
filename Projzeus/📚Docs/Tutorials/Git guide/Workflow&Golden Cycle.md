#### Шаг 1: Подготовка (Pull)

**Что делать:** Перед тем как начать писать код, нажми **Git: Pull**. **Зачем:** Это скачивает изменения, которые мог внести твой друг, пока ты спал. Если ты этого не сделаешь и начнешь писать поверх старого кода, при пуше возникнет ошибка.

#### Шаг 2: Фиксация (Atomic Commit)

**Что делать:** Не фиксируй всё разом. Сделал одну фичу (например, поправил текст в PWA) — закоммить её одну.

- В VS Code нажми на иконку «Source Control» (три точки соединенные линиями).
    
- Нажми на **`+`** (Stage Changes) только у тех файлов, которые относятся к этой задаче.
    
- Напиши осмысленное сообщение: `feat: поправил склонения в бонусах` или `fix: кот перестал улетать за экран`. **Зачем:** Если через неделю окажется, что новая функция ломает сайт, ты сможешь отменить только этот маленький "кирпичик", а не всю работу за день.
    

#### Шаг 3: Синхронизация (Push)

**Что делать:** В конце рабочего блока нажми **Git: Push**. **Зачем:** Отправляет твои локальные "кирпичики" (коммиты) в облако (GitHub), чтобы друг мог их увидеть.

|**Команда**|**Что делает (на человеческом)**|**Аналог в интерфейсе**|
|---|---|---|
|`git pull`|"Забери всё новое из облака"|Кнопка Pull / Обновить|
|`git status`|"Что я вообще накодил и не сохранил?"|Список файлов в Source Control|
|`git add .`|"Приготовь ВСЕ файлы к сохранению"|Плюсик сверху (Stage All)|
|`git commit -m "сообщение"`|"Запечатай коробку и подпиши её"|Поле ввода текста + Commit|
|`git push`|"Отправь все запечатанные коробки в облако"|Кнопка Push / Sync|
