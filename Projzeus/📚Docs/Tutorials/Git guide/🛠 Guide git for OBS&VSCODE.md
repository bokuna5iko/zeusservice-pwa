
## 1. Начало работы (Синхронизация)

- **`git checkout main`** — перейти на главную ветку.
    
- **`git checkout pwa-update`** — перейти на твою основную рабочую ветку.
    
- **`git fetch origin`** — проверить, не создал ли друг новые ветки на GitHub (чтобы они появились в списке).
    
- **`git pull`** — скачать обновления из облака. **Делай это всегда перед началом работы.**
    

## 2. Сохранение и отправка (Твой цикл)

- **`git add .`** — подготовить все файлы к сохранению.
    
- **`git commit -m "описание"`** — создать точку сохранения. Пиши осмысленные комментарии (например: `"fix: калькулятор бонусов"`).
    
- **`git push`** — отправить код на GitHub.
    
- **`git push --set-upstream origin название-ветки`** — если ветка новая, эта команда создаст её копию на GitHub.
    

## 3. Работа с ветками (Безопасность)

- **`git checkout -b feature/name`** — создать «песочницу» (новую ветку) для рискованных правок.
    
- **`git branch -a`** — список всех веток (локальных и удаленных).
    
- **`git merge название-ветки`** — влить изменения из одной ветки в ту, в которой ты сейчас.
    
- **`git merge --abort`** — «Кнопка отмены»: если при слиянии всё пошло не так (куча конфликтов), эта команда вернет всё как было до начала мерджа.
    

## 4. Чистка и исправление ошибок

- **`git status`** — самая полезная команда. Скажет, в какой ты ветке и что нужно сделать прямо сейчас.
    
- **`git checkout .`** — сбросить все незакоммиченные изменения (если запутался в коде и хочешь начать сначала).
    
- **`git log -n 5`** — посмотреть последние 5 сохранений (коммитов). Нажми `q`, чтобы выйти.
    
- **`git stash`** — быстро «спрятать» свои недоделанные правки в карман, чтобы переключиться на другую ветку.
    
- **`git stash pop`** — достать спрятанные правки обратно.
    

## 5. Продвинутые команды (Для сложных случаев)

- **`git rm -r --cached .obsidian`** — заставить Git «забыть» про папку Obsidian (но оставить её на диске).
    
- **`git merge --allow-unrelated-histories`** — слить ветки, если Git ругается на «разные истории» (как было сегодня).
    
- **`git clone [ссылка] .`** — скачать проект в текущую пустую папку.

### Шпаргалка по редактору Vim (Черный экран)

Если ты попал в «черный экран» после мерджа:

1. Нажми **`Esc`**.
    
2. Введи **`:wq`** и **`Enter`** (сохранить и выйти).
    
3. Введи **`:q!`** и **`Enter`** (выйти без сохранения).
    

> **Совет:** Если терминал «завис» (внизу написано `(END)` или `:`), просто нажми **`q`**, чтобы вернуться к вводу команд.
