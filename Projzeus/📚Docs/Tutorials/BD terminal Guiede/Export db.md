

⚠️ **Важно перед запуском:** Все команды на твоем ПК выполняются строго после перехода в папку утилит Postgres в консоли `cmd`:

DOS

```
cd "C:\Program Files\PostgreSQL\18\bin"
```

## Сценарий 1. Из БД VPS ➡️ на локальную БД (Скачать и обновить локалку)

Этот скрипт забирает актуальную базу с сервера и накатывает её тебе на комп.

### Шаг А. Создаем дамп с VPS прямо на диск C:\

DOS

```
pg_dump.exe --host=10.16.0.4 --port=5432 --username=zeus_user --format=p --encoding=UTF-8 --file=C:\zeus_from_vps.sql zeus_auto_db
```

_(Затребует пароль от удаленного `zeus_user`)._

### Шаг Б. Заливаем этот файл в свою локальную базу

_(Перед этим не забудь удалить старые таблицы в локальном pgAdmin, чтобы связи не конфликтовали!)_

DOS

```
psql -U postgres -d zeus_db -f "C:\zeus_from_vps.sql"
```

_(Затребует пароль от твоего локального `postgres`)._

## Сценарий 2. Из локальной БД ➡️ на БД в VPS (Залить локальные изменения на сервер)

Этот сценарий делает ровно обратное: собирает то, что ты наворотил на компе, и отправляет на сервер.

### Шаг А. Делаем дамп локальной базы на диск C:\

DOS

```
pg_dump.exe -U postgres -d zeus_db -F p -f "C:\zeus_to_vps.sql"
```

_(Затребует пароль от твоего локального `postgres`)._

### Шаг Б. Заливаем локальный файл напрямую на VPS через VPN

_(Перед этим не забудь очистить папку Tables на сервере через DBeaver, чтобы не было ошибок "already exists"!)_

DOS

```
psql -h 10.16.0.4 -p 5432 -U zeus_user -d zeus_auto_db -f "C:\zeus_to_vps.sql"
```

_(Затребует пароль от удаленного `zeus_user`)._

### 💡 Шпаргалка по переменным:

- **`10.16.0.4`** — IP твоего сервера внутри VPN.
    
- **`zeus_auto_db`** — имя базы данных на сервере VPS.
    
- **`zeus_user`** — пользователь БД на сервере.
    
- **`zeus_db`** — имя твоей локальной базы в pgAdmin.
    
- **`postgres`** — твой локальный суперпользователь.
