## 1. Правило контекста: Роль + Задача + Формат (Промптинг)

Локальная модель — это чистый лист. Каждое новое окно чата она начинает «с амнезией». Если ты просто напишешь: _«Как сделать выпадающее меню?»_, она выдаст банальный абстрактный код.

Чтобы получить идеальный ответ, конструируй промт по формуле:

1. **Кто она (Роль):** `Ты опытный фронтенд-разработчик и терпеливый ментор.`
    
2. **Что у тебя есть (Контекст):** `Я пишу проект на React + Vite, использую чистый CSS (dark theme).`
    
3. **Что нужно сделать (Задача):** `Напиши компонент выпадающего меню (Dropdown).`
    
4. **В каком виде выдать (Формат):** `Выдай код компонента и отдельно CSS-стили. Ключевые строчки прокомментируй.`
    

## 2. Разделяй сложные задачи на итерации (Step-by-Step)

Не проси модель написать «весь сайт» или «целую админку» одним запросом. Она запутается, начнет галлюцинировать или оборвет код на середине.

- **Плохо:** _«Напиши мне систему лояльности для автомойки с базой данных и сканером QR-кодов»._
    
- **Правильно:**
    
    - _Шаг 1:_ «Помоги спроектировать структуру таблицы в PostgreSQL для хранения бонусов клиентов автомойки».
        
    - _Шаг 2:_ «Отлично, таблица есть. Теперь напиши SQL-запрос, который начисляет клиенту 10% баллов от суммы чека».
        
    - _Шаг 3:_ «Теперь давай напишем функцию на JS, которая будет генерировать строку для QR-кода на основе ID клиента».
        

## 3. Критическое мышление (Модели могут «галлюцинировать»)

Помни: локальная модель не проверяет код в терминале. Она просто угадывает, какие символы должны идти друг за другом.

- **Всегда перепроверяй её код.** Если она предлагает использовать метод библиотеки, которого ты никогда не видел — спроси её прямо: _«Этот метод точно существует в этой версии библиотеки, или ты его придумал?»_
    
- Если код выдает ошибку, не пытайся исправить её сам вручную, если не понимаешь суть. Просто скопируй текст ошибки из консоли, вернись в чат и скажи: _«Твой код выдал вот такую ошибку: [ВСТАВИТЬ ОШИБКУ]. Почему она возникла и как исправить?»_ Модель моментально перестроится.
    

## 4. Используй её как ментора, а не как генератор шпаргалок

Если ты будешь просто слепо копировать код из LM Studio в свой проект, ты ничему не научишься. Используй Qwen для понимания **сути**.

Задавай ей «почемучкиные» вопросы:

- _«Ты использовал здесь стрелочную функцию. Почему именно её, а не обычную через слово `function`?»_
    
- _«Объясни мне эту строчку кода так, будто мне 10 лет»._
    
- _«Какие есть минусы у того подхода, который ты только что предложил?»_
    

## 5. Управляй окном контекста («Чисти за собой»)

Поскольку модель работает локально на твоем ПК, она держит в памяти всю историю текущей переписки.

- Если чат тянется слишком долго (ты обсудил уже 5 разных тем), модель начнет подтупливать, путаться или медленнее отвечать.
    
- **Заведи привычку:** Одна задача — одна вкладка чата. Решил вопрос с сортировкой массива? Отлично, создавай новый пустой чат для обсуждения стилей CSS. Так модель всегда будет работать на максимальной скорости и с "свежей головой".
