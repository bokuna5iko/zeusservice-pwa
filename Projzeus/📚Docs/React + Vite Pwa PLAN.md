### Этап 1: Подготовка окружения

На этом этапе мы создадим новый каркас проекта рядом с текущим, чтобы ничего не потерять.

1. **Инициализация Vite:** Развернем проект с шаблоном `react` (используем JS или TS по твоему желанию).
    
2. **Чистка структуры:** Удалим лишние файлы от Vite (логотипы, дефолтные стили), чтобы они не мешались.
    
3. **Перенос статики:** Переместим иконки, манифест и изображения из твоей папки `public` в новый проект.
    
4. **Установка зависимостей:** Добавим `axios` (для запросов к API) и `react-router-dom` (для навигации между экранами сканера, профиля и админки).
    

### Этап 2: Настройка PWA и Стилизации

Сделаем так, чтобы приложение сразу «чувствовало» себя как мобильное.

1. **Конфигурация `vite-plugin-pwa`:** Настроим автоматическую генерацию Service Worker. Это заменит твой ручной `sw.js`.
    
2. **Импорт CSS:** Подключим твои готовые стили (`base.css`, `layout.css`, `pages.css`). React позволяет использовать их практически без правок.
    

### Этап 3: Создание ядра (Shared Logic)

Переносим функции, которые не связаны с отрисовкой напрямую.

1. **API Layer:** Перенесем логику из твоего `api.js` в отдельные функции/хуки.
    
2. **Auth Context:** Создадим механизм, который будет «помнить» залогинен пользователь или админ на всех страницах (в React это делается через `Context API`).
    

### Этап 4: Компонентная сборка (Главный экран и Лояльность)

Самая интересная часть — превращаем HTML в живые компоненты.

1. **Layout:** Создадим общую оболочку с навигацией (твое нижнее меню).
    
2. **LoyaltyCard:** Перенесем логику отрисовки кружочков. Теперь это будет массив, который рендерится автоматически.
    
3. **User Profile:** Отображение данных пользователя (баланс, имя, история).
    

### Этап 5: Сложные модули (Сканер и Админка)

1. **Scanner Component:** Интегрируем библиотеку `html5-qrcode` внутрь React-компонента.
    
2. **Admin Panel:** Перенесем таблицы и статистику. Сделаем фильтрацию посещений более быстрой.
    

### Этап 6: Тестирование и Деплой

1. **Проверка офлайн-режима:** Убедимся, что PWA корректно кеширует страницы.
    
2. **Деплой на Vercel:** Настроим билд, чтобы при пуше в Git проект обновлялся автоматически.
