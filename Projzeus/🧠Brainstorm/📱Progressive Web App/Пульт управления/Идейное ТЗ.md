
# 📝 Идейное ТЗ: Структура «Пульта управления» администратора Zeus Auto (Версия 2.0)

## 1. Логика Смен и Календаря (Глобальный контекст)

- **Режим работы:** Строго ежедневно с 09:00 до 22:00.
    
- **Связь с базой:** Все данные жестко привязаны к календарным датам. В 09:00 админ нажимает кнопку «Открыть смену» (подтягивается текущая дата), после чего активируются вкладки управления. В 22:00 смена закрывается, фиксируя финальный финансовый результат дня и блокируя редактирование.
    

## 2. Логика Адаптивности и Устройств (Умный роутинг)

Система автоматически подстраивает интерфейс под тип устройства администратора (без создания лишних ролей):

- **Доступ с телефона (Экран < 1024px):** Админ видит стандартное мобильное PWA-приложение автомойки (сканер QR-кодов, текущий профиль, базовые действия). Плотный табличный пульт скрыт во избежание визуальной каши.
    
- **Доступ с ПК / Планшета (Экран >= 1024px):** Автоматически открывается десктопный полноэкранный «Пульт управления» с трехсекционной структурой вкладок.
    

## 3. Архитектура Вкладок Пульта Управления (на ПК / Планшете)

### 📂 ВКЛАДКА 1: Лента визитов (Основной рабочий экран)

- **Назначение:** Быстрый ввод и контроль текущего потока машин в реальном времени.
    
- **Верхняя панель:** Дашборд кассы с крупными живыми цифрами (_Всего_, _Нал_, _Безнал_, _Расходы смены_) и кнопка `[ + ДОБАВИТЬ РАСХОД ]`.
    
- **Основная таблица (Строго в утвержденном порядке):**
    
    1. **Номер визита** — порядковый номер строки за текущий день.
        
    2. **Марка автомобиля** — текстовое поле с автодополнением моделей.
        
    3. **Имя + юзернейм** — данные профиля (из PWA по QR-коду или ручной ввод).
        
    4. **Номер телефона** — телефон клиента (с маской автоформата).
        
    5. **Вид услуги** — выпадающий список из прайс-листа.
        
    6. **Стоимость услуги** — сумма, подтягиваемая из прайса (с возможностью ручной правки/скидки).
        
    7. **Номер визита в акции** — счетчик лояльности (число от 1 до 8), транслируемый из существующей базы данных клиента.
        
    8. **Способ оплаты** — переключатель (_Нал / Карта / СБП / Р.счет_).
        
- **Действия в конце строки:** Кнопка `[Изменить]` (вызов модального окна для моментального исправления ошибок в строке) и кнопка `[ X ]` (удаление/отмена заезда).
    

### 📂 ВКЛАДКА 2: Работа с сотрудниками

- **Назначение:** Управление персоналом в рамках текущего дня.
    
- **Функционал:**
    
    - _Учет выходов на работу:_ Отметка чекбоксами или тегами, кто из мойщиков сегодня заступил на смену (например, `[✓] Андрей`, `[ ] Антон`).
        
    - _Статистика выработки за день:_ Компактная таблица, показывающая количество и список машин, помытых каждым сотрудником за 13-часовую смену, и автоматический расчет его заработка за день на основе внутренних процентов.
        

### 📂 ВКЛАДКА 3: Мобильная админка (Симулятор)

- **Назначение:** Полный доступ к мобильному функционалу админа прямо с экрана компьютера.
    
- **Функционал:** Внутри вкладки рендерится аккуратный фрейм/контур смартфона. Внутри этого фрейма отображается стандартная мобильная версия админки. Это позволяет админу выполнять любые «телефонные» операции (просмотр профилей, истории, начислений в мобильном формате), не вставая из-за ПК и не переключая устройства.
