## Итоговое дизайнерское ТЗ (Техническое задание) под ключ

_Для PWA-приложения ZEUS AUTO (Разделы: Главная и История)_

### 1. Цветовая система интерфейса (Правило 3-х цветов)

- **Базовый фон (80%):** Глубокий темный (сине-черный) для обеспечения комфорта водителя в темное время суток и снижения нагрузки на экран.
    
- **Брендовый / Интерактивный (15%):** Неоновый синий/голубой. Используется для обозначения активных действий, кнопок, кликабельных элементов и пройденного прогресса.
    
- **Акцентный (5%):** * _Золотой:_ Строго для обозначения сверхвыгоды (VIP-пакеты, карточки акций, иконка доступного подарка).
    
    - _Зеленый:_ Строго для обозначения финансовой выгоды клиента (блок «Сэкономлено», бесплатные услуги, примененные скидки).
        

### 2. Главный экран: Оптимизация UX и логики

- **Экран смартфона (Экономия места):** Удалить дублирующий статичный блок «ВИЗИТОВ: 5» в самом низу экрана. Высвободившееся пространство использовать для «воздуха» или поднятия контента вверх.
    
- **Интерактив QR-кода:** При клике на QR-код он плавно увеличивается в размерах по центру экрана, а весь остальной фон приложения затемняется (Overlay). Экран смартфона принудительно переходит в режим максимальной яркости (если реализуемо в рамках PWA) для безошибочного считывания сканером кассира.
    
- **Визуализация таймера QR:** Геометрическую рамку вокруг кода («Букву Г») адаптировать под интуитивно понятный формат: добавить классический убывающий линейный прогресс-бар или цифровой счетчик времени жизни кода (например, `0:59`), чтобы пользователь понимал суть анимации.
    
- **Выделение пакетов услуг:**
    
    - _Пакет «10 моек» (Маркетинговый лидер):_ Сохраняет золотое позиционирование. Вместо плотной заливки карточки использовать глубокий фоновый цвет приложения с добавлением **яркой золотой неоновой рамки (border)** и мягкого внутреннего свечения. Старая цена — зачеркнута.
        
    - _Пакет «Комплексная» (Стандарт):_ Сделать контур карточки чуть светлее основного фона, добавить неоново-синий акцент на иконку, чтобы обозначить кликабельность элемента.
        

### 3. Система лояльности (Прогресс-бар)

- **Тип отображения:** Линейный таймлайн с контрольными точками (**Milestone Track**).
    
- **Логика отображения:** * Линия прогресса вытягивается горизонтально. Пройденные шаги (1, 2, 3...) загораются ярким неоновым синим цветом.
    
    - Промежуточная контрольная точка (4-й визит) выделяется размером и имеет несмываемую подпись «20%».
        
    - Будущие пустые шаги (5, 6, 7) отображаются как лаконичные темные точки/засечки на линии без текстовых цифр, чтобы не перегружать интерфейс.
        
    - Финальная точка (8-й визит) — иконка подарка.
        

### 4. Экран «История»: Чистка шума и логика статусов

- **Блок «Сэкономлено»:** Сделать визуально доминантным по отношению к блоку «Потрачено». Зеленый шрифт суммы экономии сделать крупнее.
    
- **Логика зеленых плашек (Выгода):**
    
    - Если услуга платная, но со скидкой: цена пишется белым, на зеленой плашке пишется `-600 ₽` или `Скидка 600 ₽`.
        
    - Если услуга бесплатная (8-й визит): вместо цены пишется крупно зеленым `БЕСПЛАТНО` (или `0 ₽`), на плашке указывается `Подарок`.
        

### 5. Правила анимации и микро-взаимодействий (UX без тормозов)

- **Отказ от жестов:** Полностью исключить анимацию свайпа страниц пальцем (влево/вправо) во избежание конфликтов с навигацией браузера/ОС.
    
- **Переключение страниц:** Использовать системную нативную анимацию **Fade + Shift** (плавное растворение текущего экрана с микро-сдвигом вверх за 150–200 мс и появление нового).
    
- **Успешный скан QR:** **Запретить анимацию конфетти на рядовой скан.** Вместо нее использовать микро-отклик: QR-код плавно трансформируется в аккуратную зеленую галочку ($\checkmark$) с короткой тактильной вибрацией смартфона (haptic feedback). Конфетти допускается _только один раз_ — в момент, когда пользователь нажимает кнопку «Забрать подарок» на 8-м визите.
    
- **Анимация подарка:** Для привлечения внимания к финальной цели лояльности использовать эффект **«Дыхания» + Золотой блик**. Иконка подарка циклично и плавно увеличивается/уменьшается на 4-5% (период 2-3 сек), а раз в 5 секунд по ней пробегает мягкий золотистый градиентный блик (shimmer).
    
- **Появление списка истории:** Карточки истории загружаются с микро-задержкой друг за другом снизу вверх, создавая «эффект домино» (Staggered animation).
    
- **Раскрытие деталей:** При клике на карточку истории нижние блоки плавно сдвигаются вниз, открывая скрытую область по принципу «аккордеона».
