# 🧠 Business Logic (Buslog) — Логика приложения

Этот раздел описывает, как работает "внутренняя кухня" PWA: расчеты, условия и переключение состояний.

## 🧼 Система лояльности "8+1"

Основная механика: клиент копит посещения. На 8-й визит он получает уведомление, что следующий (9-й) — бесплатный.

### Алгоритм:
1. **Счетчик (counter):** Целое число от 0 до 8.
2. **Триггер "Поздравление":** - Если `counter == 8`, включается состояние `isNextFree = true`.
   - Визуально: карточка подсвечивается золотым/зеленым, вылетает плашка "Ваша следующая мойка БЕСПЛАТНО!".
3. **Сброс:** После использования бесплатной мойки (нажатие кнопки админом или ботом), `counter` сбрасывается в `0`.

При первом заходе пользователя, появляется экран авторизации, где можно войти по своему номеру и войти в аккаунт, или же создать аккаунт и далее войти в аккаунт.

### 🔵 Сценарий 2: Отправка данных калькулятора (POST /order)

_Это то, что бэкэнд присылает в ответ после того, как админ нажал «Подтвердить»._

```json

     "grossAmount": 600,

     "appliedBonus": true 
}, 
   "newStatus": { 
     "visitCount": 0, 
     "isEligibleForFreeWash": false 
  } 
}
```

^order-json

#### 1. Процесс сканирования (Интерфейс Админа)

Когда камера считывает QR:

- **Действие:** Запрос `GET /user?id={userId}`.
    
- **Обработка:** Мы получаем объект со статусом (`visitCount`, `isEligibleForFreeWash`).
    
- **UI:** На экране админа загорается карточка клиента. Если `isEligibleForFreeWash: true`, интерфейс калькулятора должен подсветиться (например, золотой рамкой), чтобы админ сразу видел: «Ого, сейчас будет бонус!».

#### 2. Процесс в Калькуляторе

Здесь вступает в дело та самая «двойная цена»:

- **Логика:** При выборе услуги (например, «Стандарт») программа проверяет текущий `visitCount`.
    
- **Расчет:**
    
    - `grossAmount` = всегда берется из справочника (600 ₽).
        
    - `netAmount` = если `visitCount == 7`, то автоматически выставляем **0 ₽**, иначе — **600 ₽**.
        
- **Визуал:** Админ видит: «К оплате: 0 ₽ (Бонусная мойка)». Это исключает человеческий фактор — кассир не забудет про акцию.

#### 3. Закрытие сделки

- **Действие:** Нажатие кнопки «Подтвердить».
    
- **Обработка:** Отправка `POST /order`.
    
- **Результат:** В ответ прилетает `newStatus` с `visitCount: 0`. Твой интерфейс обновляется, и админ видит, что бонус списан успешно.
