# 📊 Metrics & Analytics (v0.1)

> [!abstract] Суть
> Этот файл описывает, какие данные мы собираем с каждого заказа для формирования отчетов Боссу.

---

## 1. Структура транзакции (что летит в БД)
При каждом закрытии чека в Калькуляторе, сервер сохраняет:
- `serviceId`: Какая мойка была выбрана.
- `grossAmount`: Цена по прайсу (например, 600₽).
- `netAmount`: Реально полученные деньги (0₽, если бонус, или 600₽).
- `appliedBonus`: (true/false) — был ли это 8-й подарок.

---

## 2. Ключевые показатели для Босса
Эти цифры будут выводиться в админ-панели:

1. **Оборот (Potential Revenue):** Сумма всех `grossAmount`. (Сколько бы заработали без скидок).
2. **Чистая касса (Actual Cash):** Сумма всех `net_amount`. (Сколько денег реально в сейфе).
3. **Убыток на лояльность (Cost of Loyalty):** Разница между Оборотом и Кассой. (Цена привлечения клиентов).
4. **Популярность услуг:** Статистика по `serviceId` (что чаще выбирают: Стандарт или Комплекс).
