## 📝 Журнал обновлений (03.05.2026)

### 🚀 Backend & API (Refactoring)
В рамках синхронизации с бизнес-логикой из Obsidian и улучшения архитектуры были внесены следующие изменения:

*   **Middleware `authenticateToken`**: Реализована централизованная проверка JWT-токенов. Теперь данные пользователя (ID и роль) доступны в `req.user` для всех защищенных маршрутов.
*   **Синхронизация профиля**: Эндпоинт `/api/user/me` переведен на формат CamelCase в соответствии с фронтенд-стандартами:
    *   `total_visits` → `visitCount`
    *   `last_visit` → `lastVisitDate`
*   **Автоматизация бонусов**: Расчет бесплатной мойки перенесен на бэкенд. API теперь возвращает:
    *   `isEligibleForFreeWash`: `true`, если визит кратен 8.
    *   `nextBonusIn`: количество моек, оставшихся до следующего подарка.

### 🎨 Frontend & UI
*   **Интеграция с новым API**: Обновлены методы в `api.js` и `main.js` для работы с расширенным объектом профиля.
*   **Логика прогресс-бара**: Шкала баллов теперь динамически подтягивает данные из БД. При достижении 8 визитов интерфейс автоматически подсвечивает подарок.
*   **Безопасность**: Добавлена автоматическая переадресация на страницу авторизации при истечении срока действия токена.
